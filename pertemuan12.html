Laporan Klasifikasi Menggunakan Random Forest di KNIME#

1. Judul#

Klasifikasi Data Tanah Menggunakan Metode Random Forest pada Aplikasi KNIME

2. Tujuan#

Tujuan dari tugas ini adalah untuk melakukan proses klasifikasi data menggunakan algoritma Random Forest di KNIME. Model klasifikasi dibuat untuk memprediksi kolom target berdasarkan beberapa atribut atau fitur yang terdapat pada dataset.

3. Dataset#

Dataset yang digunakan berisi data karakteristik tanah. Beberapa atribut yang digunakan sebagai fitur dalam proses klasifikasi antara lain:

  • pH Tanah

  • N Total (%)

  • P Tersedia (ppm)

  • K Tersedia (meq/100g)

  • C Organik (%)

  • KTK (meq/100g)

  • Kejenuhan Basa (%)

  • Tekstur Tanah

  • Kadar Air (%)

Kolom ID tidak digunakan dalam proses klasifikasi karena hanya berfungsi sebagai identitas data dan tidak berpengaruh terhadap proses prediksi.

Kolom target yang digunakan adalah:

  • Label

Kolom Label merupakan kelas atau kategori yang akan diprediksi oleh model Random Forest.

4. Metode yang Digunakan#

Metode yang digunakan dalam tugas ini adalah Random Forest.

Random Forest adalah algoritma machine learning yang termasuk ke dalam metode ensemble learning. Random Forest bekerja dengan membangun banyak decision tree, kemudian menggabungkan hasil prediksi dari setiap tree untuk menghasilkan keputusan akhir.

Pada klasifikasi, Random Forest menggunakan metode voting. Kelas yang memperoleh suara terbanyak dari beberapa decision tree akan menjadi hasil prediksi akhir.

5. Alur Workflow di KNIME#

Workflow klasifikasi yang dibuat di KNIME terdiri dari beberapa node, yaitu:

  1. Excel Reader

  2. Column Filter

  3. Missing Value

  4. Table Partitioner

  5. Random Forest Learner

  6. Random Forest Predictor

  7. Scorer

Alur prosesnya adalah sebagai berikut:

Excel Reader digunakan untuk membaca dataset dari file Excel. Setelah itu, Column Filter digunakan untuk memilih kolom yang diperlukan dan menghapus kolom yang tidak digunakan, seperti ID. Missing Value digunakan untuk menangani data kosong. Selanjutnya, Table Partitioner digunakan untuk membagi data menjadi data training dan data testing. Data training digunakan pada Random Forest Learner untuk membangun model, sedangkan data testing digunakan pada Random Forest Predictor untuk melakukan prediksi. Hasil prediksi kemudian dievaluasi menggunakan Scorer.

6. Penjelasan Setiap Node#

6.1 Excel Reader#

Node Excel Reader digunakan untuk membaca dataset yang berbentuk file Excel. Dataset yang sudah dibaca akan ditampilkan dalam bentuk tabel di KNIME.

6.2 Column Filter#

Node Column Filter digunakan untuk memilih kolom yang akan digunakan dalam proses klasifikasi. Pada tahap ini, kolom ID dikeluarkan karena tidak dibutuhkan dalam proses prediksi.

Kolom yang digunakan sebagai fitur antara lain pH Tanah, N Total, P Tersedia, K Tersedia, C Organik, KTK, Kejenuhan Basa, Tekstur Tanah, dan Kadar Air.

6.3 Missing Value#

Node Missing Value digunakan untuk menangani nilai kosong atau missing value pada dataset. Jika terdapat data kosong pada kolom numerik, maka dapat diisi menggunakan nilai rata-rata atau mean. Jika terdapat data kosong pada kolom kategori, maka dapat diisi menggunakan nilai yang paling sering muncul.

6.4 Table Partitioner#

Node Table Partitioner digunakan untuk membagi dataset menjadi dua bagian, yaitu data training dan data testing.

Pengaturan yang digunakan adalah:

  • Relative size: 70%

  • Sampling strategy: Stratified

  • Group column: Label

  • Random seed: 42

Pembagian data dilakukan dengan rasio 70:30, yaitu 70% data digunakan untuk training dan 30% data digunakan untuk testing. Stratified sampling digunakan agar proporsi kelas pada data training dan testing tetap seimbang.

6.5 Random Forest Learner#

Node Random Forest Learner digunakan untuk melatih model klasifikasi menggunakan algoritma Random Forest.

Pengaturan yang digunakan adalah:

  • Target column: Label

  • Attribute selection: Use column attributes

  • Split criterion: Gini Index atau Information Gain Ratio

  • Number of trees / models: 100

  • Maximum depth: 10

  • Random seed: 42

Target column yang digunakan adalah Label, karena kolom tersebut merupakan kelas yang ingin diprediksi.

6.6 Random Forest Predictor#

Node Random Forest Predictor digunakan untuk melakukan prediksi terhadap data testing. Node ini menerima dua input, yaitu model dari Random Forest Learner dan data testing dari Table Partitioner.

Hasil dari node ini berupa kolom baru, yaitu Prediction (Label), yang menunjukkan hasil prediksi model terhadap data testing.

6.7 Scorer#

Node Scorer digunakan untuk mengevaluasi hasil prediksi model.

Pengaturan pada Scorer adalah:

  • First column: Label

  • Second column: Prediction (Label)

Label merupakan kelas asli, sedangkan Prediction (Label) merupakan kelas hasil prediksi dari model Random Forest.

7. Parameter Random Forest#

Beberapa parameter penting yang digunakan dalam Random Forest adalah sebagai berikut:

7.1 n_estimators#

n_estimators adalah jumlah decision tree yang digunakan dalam Random Forest. Pada tugas ini digunakan 100 tree.

Semakin banyak jumlah tree, hasil prediksi biasanya semakin stabil. Namun, waktu komputasi juga akan semakin lama.

7.2 max_depth#

max_depth adalah kedalaman maksimal setiap decision tree. Pada tugas ini digunakan nilai 10.

Jika nilai max_depth terlalu kecil, model dapat mengalami underfitting. Jika nilai max_depth terlalu besar, model dapat mengalami overfitting.

7.3 Split Criterion#

Split criterion adalah metode yang digunakan untuk menentukan pembagian data pada setiap node decision tree. Contoh split criterion yang dapat digunakan adalah Gini Index atau Information Gain Ratio.

8. Hasil Evaluasi#

Hasil evaluasi diperoleh dari node Scorer. Evaluasi model dapat dilihat melalui confusion matrix, accuracy, dan error rate.

Confusion matrix menunjukkan jumlah data yang berhasil diklasifikasikan dengan benar dan jumlah data yang salah diklasifikasikan.

Accuracy menunjukkan tingkat ketepatan model dalam melakukan klasifikasi. Semakin tinggi nilai accuracy, maka semakin baik performa model.

Error rate menunjukkan tingkat kesalahan model dalam melakukan klasifikasi. Semakin kecil nilai error rate, maka semakin baik performa model.

9. Interpretasi#

Berdasarkan workflow yang telah dibuat, model Random Forest berhasil digunakan untuk melakukan klasifikasi data tanah berdasarkan fitur-fitur yang tersedia. Data dibagi menjadi data training dan testing dengan perbandingan 70:30. Model dilatih menggunakan data training, kemudian diuji menggunakan data testing.

Hasil prediksi dievaluasi menggunakan Scorer dengan membandingkan kolom Label sebagai data asli dan Prediction (Label) sebagai hasil prediksi. Jika nilai accuracy tinggi, maka model dapat dikatakan memiliki kemampuan klasifikasi yang baik.

10. Kesimpulan#

Berdasarkan proses klasifikasi yang dilakukan di KNIME, dapat disimpulkan bahwa algoritma Random Forest dapat digunakan untuk membangun model klasifikasi pada data tanah. Random Forest bekerja dengan menggabungkan banyak decision tree sehingga hasil prediksi menjadi lebih stabil.

Workflow yang digunakan terdiri dari proses pembacaan data, pemilihan kolom, penanganan missing value, pembagian data, pelatihan model, prediksi, dan evaluasi. Parameter yang digunakan antara lain jumlah tree sebanyak 100 dan kedalaman pohon maksimal sebesar 10.

Model dievaluasi menggunakan Scorer dengan membandingkan data asli dan hasil prediksi. Hasil evaluasi dapat digunakan untuk mengetahui seberapa baik model dalam melakukan klasifikasi.
